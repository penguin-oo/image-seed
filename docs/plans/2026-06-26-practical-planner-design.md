# Image Seed Practical Planner Design

## Goal

将 Image Seed 从“单关键词课程演示”升级为有实际使用价值的摄影项目策划工具。用户输入主题、地点、对象、器材、风格、周期和交付形式后，系统输出可执行的拍摄计划、镜头清单、风险提醒和参考资料。

## Approach Options

1. **Recommended: 本地实用策划器 + Coze 大模型工作流设计**
   - 本地网页不写 API Key，保证稳定部署。
   - UI 展示完整多字段输入和可执行输出。
   - Coze 文档和配置明确使用扣子自带大模型节点，登录后可在画布中复现。

2. **只做 Coze 工作流**
   - 优点是 AI 真实感更强。
   - 缺点是受登录、验证码、平台权限影响，公网作业网页不可稳定演示。

3. **前端直接接 API**
   - 不符合当前“不要 API”的要求，也会暴露 Key 或依赖后端。

## Selected Design

采用方案 1。前端作为可提交、可访问、可演示的完整项目；Coze 作为课程要求中的工作流和大模型平台设计。报告中明确扣子自带大模型节点用于真实工作流，网页端用同结构本地规则模拟。

## User Input

- 摄影主题：必填
- 拍摄地点：校园、街区、家中、老小区、商业空间等
- 拍摄对象：人物、空间、物件、街头瞬间、档案材料
- 器材：手机、微单、胶片机、黑白胶片等
- 风格：纪实、街头、观念、家庭档案、类型学等
- 周期：1 天、3 天、1 周、2 周
- 交付形式：课程作业、摄影书、展览墙、作品集

## Output

- 项目定位和创作概念
- 拍摄策略
- 分阶段日程
- 12 张成片镜头清单
- 场景、道具、光线建议
- 风险提醒
- 摄影师推荐
- 摄影书推荐
- 素材参考墙
- 收藏和历史记录

## Coze Design

Coze 工作流使用扣子自带大模型节点，不在网页中写外部 API。节点链路：

开始节点 -> 大模型分析节点 -> 摄影师数据库查询 -> 摄影书数据库查询 -> 拍摄计划整合 -> 历史记录写入 -> 输出节点

## Visual Design

保留暗房/摄影研究工作台气质，但首页从大字海报改成“实际填写表单 + 右侧视觉素材墙”。结果页强调可执行计划，减少空泛文案，增加清单和时间表。
